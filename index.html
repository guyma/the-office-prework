העובד הראשון שלך 0/9 הושלמו

מגיעים מוכנים לסדנה

תשע משימות לפני שנפגשים. עובדים ממחשב, לא מטאבלט, ועל המחשב שאיתו תגיעו לסדנה. כשהן מאחוריכם, שלוש השעות בזום הולכות על בנייה, לא על התקנות. מסמנים כל משימה שהושלמה, והדף זוכר אתכם. וחשוב: ההתקנות לוקחות זמן. תתחילו כמה ימים לפני הסדנה, לא בערב שלפני.

למה ההכנה חשובה

בסדנה אתם בונים עובד AI על העסק האמיתי שלכם. בשביל זה הכלים צריכים להיות מותקנים, והחומרים של העסק צריכים להיות ביד.

לפני הסדנהמתקינים, בודקים, וממלאים שאלון קצר. הכול פה בדף.
בסדנהבונים את העובד הראשון שלכם, על העסק שלכם.
אחרי הסדנהממשיכים לבנות בקצב שלכם, והעובד כבר מכיר אתכם.

חשבון Claude בתשלום

כ-5 דקות

העובד שלכם רץ על Claude Code, וזה דורש מנוי בתשלום. מתחילים עם Pro, ‏20$ לחודש. למי שבא לעבוד עם העובד ברצינות, ההמלצה שלי: Max. אפשר תמיד לשדרג אחר כך.

  • נכנסים אל claude.ai ונרשמים.
  • בוחרים מסלול בתשלום (Pro ומעלה). המסלול החינמי לא כולל את Claude Code.

איך יודעים שעבד: בהגדרות החשבון רואים Pro או Max.

מתקינים את Git

כ-3 דקות
  • מורידים Git for Windows מהעמוד הרשמי: git-scm.com
  • מריצים את ההתקנה (אם מופיעה אזהרת מפרסם: החותם הרשמי של Git for Windows הוא Johannes Schindelin; שם אחר? עוצרים ושואלים). יופיעו הרבה מסכי אפשרויות, לוחצים Next בכולם. ברירות המחדל טובות.
  • בסיום סוגרים ופותחים את Claude, כדי שיזהה את Git.
  • ברוב מחשבי המק Git כבר מגיע מובנה, אז קודם בודקים: פותחים את אפליקציית Terminal (מחפשים ב-Spotlight), מקלידים git --version ולוחצים Enter.
  • הופיע מספר גרסה? מותקן, סיימתם. קפץ חלון שמציע להתקין? לוחצים Install ומחכים שיסיים.

איך יודעים שעבד: במק, git --version מציג מספר גרסה. ב-Windows האימות האמיתי הוא משימה 7: אם הבדיקה שם עוברת, Git תקין.

הבדיקה

כ-5 דקות

עכשיו הרגע הראשון שלכם עם העובד. חמש דקות, בלי להתחייב:

  • יוצרים במחשב תיקייה חדשה וריקה, קוראים לה test.
  • פותחים את Claude, נכנסים ללשונית Code, בוחרים Local, לוחצים Select folder ובוחרים את התיקייה.
  • כותבים בתיבה: צור קובץ בשם hello.md ובתוכו ברכת שלום קצרה בעברית.
  • קלוד יציג את מה שהוא רוצה לכתוב ויחכה לאישור שלכם. זה לא באג, ככה עובדים איתו: לוחצים Accept.
  • פותחים את התיקייה ובודקים: נוצר קובץ? פתחו אותו.

רואים את הברכה? המערכת שלכם עובדת. זה, בגדול, בדיוק מה שנעשה בסדנה, רק על העסק שלכם.

איך יודעים שעבד: קובץ hello.md קיים בתיקייה ויש בו טקסט.

תיקיית הדוגמאות

כ-15 דקות

פותחים תיקייה אחת במחשב ואוספים אליה כל דבר שאתם כתבתם. מהדוגמאות האלה העובד לומד את הקול שלכם. לפחות 3, אין תקרה:

  • פוסטים שפרסמתם: פייסבוק, אינסטגרם, לינקדאין.
  • הצעת מחיר ששלחתם.
  • שיחות וואטסאפ עם לקוחות (באפליקציה: ייצוא צ'אט).
  • מיילים ללקוחות, ניוזלטר, סיכום פגישה.
  • וגם קבצים של העסק: מחירון, חוזה, ההודעה שאתם שולחים שוב ושוב.

לא צריך לסדר כלום. אוספים, ובסדנה נלמד את העובד מהכול. שם או פרט רגיש שלא צריך לחיות בקובץ? מחליפים ל-[שם] כבר עכשיו, בזמן האיסוף.

איך יודעים שעבד: יש תיקייה, ויש בה לפחות 3 דברים.

בודקים את שם המפרסם: Anthropic, PBC (בהתקנת Claude) או Johannes Schindelin (בהתקנת Git). תואם? More info ואז Run anyway. לא תואם או לא מופיע? עוצרים וכותבים לי.

ההתקנה מבקשת אישור מנהל

במחשב פרטי, אחרי שאימתתם את המפרסם: מאשרים. מחשב של העבודה? לא עוקפים, פונים ל-IT או אליי.

האינטרנט מסונן

ברשת ביתית: נסו רשת אחרת. ברשת או מחשב של העבודה: עוברים דרך ה-IT, לא עוקפים.

לוחצים על Code ומקבלים הצעה לשדרג

סימן שהחשבון עוד לא בתשלום. חוזרים למשימה 1, משדרגים, ופותחים את האפליקציה מחדש.

משהו אחר לא מסתדר

כותבים בקבוצת הוואטסאפ או ישירות אליי. ואם לא הסתדר עד הסדנה, עולים לזום חצי שעה לפני ופותרים ביחד.
