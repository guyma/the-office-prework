העובד הראשון שלך 0/8 הושלמו

מגיעים מוכנים לסדנה

שמונה משימות לפני שנפגשים. כשהן מאחוריכם, שלוש השעות בזום הולכות על בנייה, לא על התקנות. מסמנים כל משימה שהושלמה, והדף זוכר אתכם. וחשוב: ההתקנות לוקחות זמן. תתחילו כמה ימים לפני הסדנה, לא בערב שלפני.

למה ההכנה חשובה

בסדנה אתם בונים עובד AI על העסק האמיתי שלכם. בשביל זה הכלים צריכים להיות מותקנים, והחומרים של העסק צריכים להיות ביד.

לפני הסדנהמתקינים, בודקים, וממלאים שאלון קצר. הכול פה בדף.
בסדנהבונים את העובד הראשון שלכם, על העסק שלכם.
אחרי הסדנהממשיכים לבנות בקצב שלכם, והעובד כבר מכיר אתכם.

  • מורידים את הקובץ למק: הורדה ישירה (dmg)
  • פותחים את הקובץ שירד וגוררים את Claude לתיקיית היישומים (Applications).
  • פותחים את Claude מתיקיית היישומים ומתחברים (Sign in) עם החשבון מהמשימה הקודמת. ההתחברות נפתחת בדפדפן, מאשרים וחוזרים לאפליקציה.
  • בחלק העליון של האפליקציה יש שלוש לשוניות. לוחצים על Code. מוצע לכם שדרוג מנוי? חזרו למשימה 1. מתבקשים להתחבר שוב? מתחברים, סוגרים ופותחים את האפליקציה.

מתקינים את Git

כ-3 דקות

Git שומר גרסאות של הקבצים, ו-Claude Code נשען עליו מאחורי הקלעים.

  • ב-Windows זו התקנת חובה: בלעדיה לשונית Code לא עובדת על התיקיות שלכם.
  • מורידים Git for Windows מהעמוד הרשמי: git-scm.com (הקישור הראשון, 64-bit).
  • מריצים את ההתקנה. יופיעו הרבה מסכי אפשרויות, לוחצים Next בכולם. ברירות המחדל טובות.
  • ברוב מחשבי המק Git כבר מגיע מובנה, אז קודם בודקים: פותחים את אפליקציית Terminal (מחפשים ב-Spotlight), מקלידים git --version ולוחצים Enter.
  • הופיע מספר גרסה? מותקן, סיימתם. קפץ חלון שמציע להתקין? לוחצים Install ומחכים שיסיים.

איך יודעים שעבד: ב-Windows, בתפריט Start מופיע Git. במק, git --version מציג מספר גרסה.

פותחים חשבון GitHub

כ-5 דקות

GitHub הוא הענן של עולם הקבצים: דרכו מגבים את המשרד, ובהמשך גם מקבלים עדכונים וחומרים. החשבון חינמי.

  • נכנסים אל github.com/signup ונרשמים עם המייל שלכם.
  • מאמתים את המייל (מגיע קוד לאישור).
  • שומרים את שם המשתמש והסיסמה במקום שתמצאו. נצטרך אותם בסדנה.

איך יודעים שעבד: נכנסים אל github.com ורואים שאתם מחוברים.

הבדיקה

כ-5 דקות

עכשיו הרגע הראשון שלכם עם העובד. חמש דקות, בלי להתחייב:

  • יוצרים במחשב תיקייה חדשה וריקה, קוראים לה test.
  • פותחים את Claude, נכנסים ללשונית Code, בוחרים Local, לוחצים Select folder ובוחרים את התיקייה.
  • כותבים בתיבה: צור קובץ בשם hello.md ובתוכו ברכת שלום קצרה בעברית.
  • קלוד יציג את מה שהוא רוצה לכתוב ויחכה לאישור שלכם. זה לא באג, ככה עובדים איתו: לוחצים Accept.
  • פותחים את התיקייה ובודקים: נוצר קובץ? פתחו אותו.

רואים את הברכה? המערכת שלכם עובדת. זה, בגדול, בדיוק מה שנעשה בסדנה, רק על העסק שלכם.

איך יודעים שעבד: קובץ hello.md קיים בתיקייה ויש בו טקסט.

תיקיית הדוגמאות

כ-15 דקות

פותחים תיקייה אחת במחשב ואוספים אליה כל דבר שאתם כתבתם. מהדוגמאות האלה העובד לומד את הקול שלכם. לפחות 3, אין תקרה:

  • הצעת מחיר ששלחתם.
  • שיחות וואטסאפ עם לקוחות (באפליקציה: ייצוא צ'אט).
  • מיילים ללקוחות, ניוזלטר, סיכום פגישה.
  • וגם קבצים של העסק: מחירון, חוזה, ההודעה שאתם שולחים שוב ושוב.

לא צריך לסדר כלום. אוספים, ובסדנה נלמד את העובד מהכול.

איך יודעים שעבד: יש תיקייה, ויש בה לפחות 3 דברים.
